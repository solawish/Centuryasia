- [x] 1.1 在 manifest.json 的 permissions 中加入 "storage"
- [x] 1.2 在 popup.js 定義儲存鍵與預設狀態結構（例如 STORAGE_KEY、預設欄位值）

## 2. 儲存邏輯
- [x] 2.1 實作「讀取目前表單狀態」函式（影城、電影、時間、票種、數量）
- [x] 2.2 實作「寫入目前表單狀態至 chrome.storage.local」函式
- [x] 2.3 為影城、電影、時間、票種、數量欄位綁定 change/input 事件，在變更時呼叫寫入函式

## 3. 還原邏輯
- [x] 3.1 popup 載入時非同步讀取 chrome.storage.local 取得上次儲存狀態
- [x] 3.2 若有儲存的影城且與目前影城不同，先設定影城再觸發載入電影列表；否則直接載入電影列表
- [x] 3.3 電影列表載入完成後，若儲存中有 movieProgramId 且存在於選項則還原電影；若還原了電影則觸發載入時間選項
- [x] 3.4 時間選項載入完成後，若儲存中有 timeValue 且存在於選項則還原時間
- [x] 3.5 還原票種關鍵字與數量（無需依賴動態選項）

## 4. 驗證
- [x] 4.1 手動測試：變更各欄位後關閉 popup 再開啟，確認還原正確
- [x] 4.2 手動測試：儲存的電影或時間已不存在時，確認該欄位不還原、其餘欄位仍還原
