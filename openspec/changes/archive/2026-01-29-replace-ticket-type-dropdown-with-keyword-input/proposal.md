# Change: 票種下拉改為選填關鍵字輸入框並更新訂票優先順序

## Why
目前票種為寫死的下拉選單（僅一項「999(超。盲目猜測）」），且訂票流程未使用使用者選擇，而是依程式內建優先順序（999 → 全票 → 第一個）選票。改為選填的關鍵字輸入框可讓使用者輸入票種名稱關鍵字，訂票時優先匹配該關鍵字；未輸入時改為依「全票 → 單人套票 → 第一個」的後備順序，使介面與實際選票行為一致且更彈性。

## What Changes
- 將 popup 的票種「下拉選單」改為「選填的文字輸入框」（placeholder 可提示為票種關鍵字）。
- 訂票流程中讀取該輸入框的值（可能為空）。若有值，優先選擇票種名稱**含有**該關鍵字的票種；若無符合者或輸入為空，則依序：名稱含「全票」→ 名稱含「單人套票」→ 第一個票種。
- 移除現有「優先選擇 price value 為 999」的邏輯，改為上述關鍵字／後備順序。

## Impact
- Affected specs: popup-interface, booking-flow
- Affected code: popup.html（票種表單元件）、popup.js（訂票按鈕讀取輸入值、selectTicketType 優先順序與關鍵字匹配）
